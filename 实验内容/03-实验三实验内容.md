一、实验目的
1．在实验一和实验二的基础上，进一步了解系统分析与设计的迭代过程；
2．加深对类模型、状态模型和交互模型的理解；
3．理解包图与组件图的基本概念；
4．掌握在Rational Rose、Enterprise Architect等工具中绘制对象图和包；
5．探索运用知识图谱等技术加强对智能人机交互技术的运用与体验；
6．借助轻量级知识图谱自动生成“需求-用例-类-代码-测试” 五级追踪链，快捷定位、评估任一变更的影响范围。
7．形成系统的完整的需求分析与设计及报告。
二、实验器材
1．计算机一台；
2．Rational Rose或其它UML工具软件。
3．Kimi-128k，中文输出 PlantUML， https://kimi.moonshot.cn
4．通义灵码（可选），自动生成代码，VS Code 插件市场
5．VS Code ≥1.94，IDE写文档，https://code.visualstudio.com 
6．PlantUML插件，侧边实时看图，零配置，VS Code 插件市场
7．Graphviz Preview（可选），PlantUML自动布局，同上，市场一键装
8．Neo4j AuraDB，知识图谱托管，https://console.neo4j.io

三、实验内容
运用大模型或智能体和知识图谱，进行业务建模（业务获取、业务分析）、需求建模（用户需求、系统功能确定、应用特性约定）和软件系统架构设计、构件定义等开发工作。完成文档和代码的生成、转换，同时生成各阶段工作对象与工作成果制品的管理与变更跟踪所需的知识图谱。

五、常见问题（FAQ）
Q1 没有显卡怎么办？
· 用 Kimi-128k 在线版，PlantUML 生成后拷回本地。
Q2 LLM 乱造属性？
· 先把“业务术语表”贴回 Prompt，再让它“只使用表中词汇”。
Q3 Neo4j 端口冲突？
· 改 conf/neo4j.conf  dbms.connector.bolt.listen_address=:7688
