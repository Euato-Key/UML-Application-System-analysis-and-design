# 🏥 智能候诊系统 (Smart Clinic System) - UML分析与设计实验

> **2024年秋季学期 面向对象分析与设计 实验三**  
> **小组信息：** 吴俊安、帅军锐、吴思扬、林世颖

## 📖 项目简介
本项目是针对“校医院智能候诊场景”进行的完整系统分析与设计实验。
我们利用 **LLM (Kimi/ChatGPT)** 辅助生成模型，结合 **Knowledge Graph (Neo4j)** 技术实现全链路追踪，完成了从业务获取、需求建模、系统设计到代码实现的软件工程全生命周期模拟。

## 📂 成果目录结构
本项目严格遵循实验指导书要求的五阶段流程：

*   **[`1_biz/`](./1_biz/) (阶段1 - 业务建模)**
    *   包含原始访谈录音文本、业务用例图及术语表。
    *   *核心产出：* 业务用例图、业务术语表。
*   **[`2_req/`](./2_req/) (阶段2 - 需求建模)**
    *   包含系统功能清单、核心用例详述 (MSS/Extensions)。
    *   *核心产出：* 系统级用例图、详细功能清单。
*   **[`3_design/`](./3_design/) (阶段3 - 系统设计)**
    *   包含静态结构模型（类图、包图）和动态行为模型（序列图、活动图）。
    *   *核心产出：* 核心业务活动图、领域类图。
*   **[`4_code/`](./4_code/) (阶段4 - 代码实现)**
    *   基于设计图自动生成的 SpringBoot/FastAPI 代码骨架。
    *   包含双向追踪注解 (`@trace`)。
*   **[`5_change/`](./5_change/) (阶段5 - 变更管理)**
    *   针对“预约签到”变更的影响分析报告与增量代码。
*   **[`prompt/`](./prompt/) (AIGC 资产)**
    *   存档了所有用于生成UML和代码的高质量 Prompt，可复现实验过程。

## 🛠️ 技术栈与工具链

*   **建模语言:** UML 2.0 (PlantUML)
*   **AI 辅助:** Kimi (Moonshot AI) / Cursor IDE
*   **知识图谱:** Neo4j (用于需求-代码追踪)
*   **开发环境:** VS Code + PlantUML Plugin
*   **版本控制:** Git / GitHub

## 🚀 快速开始

### 1. 查看设计图
本项目所有 UML 图均使用 `.puml` 格式编写。
*   推荐使用 **VS Code** 安装 `PlantUML` 插件。
*   打开任意 `.puml` 文件，按 `Alt + D` 即可实时预览设计图。
*   所有已渲染的图片归档在 [`out/`](./out/) 目录下。

### 2. 运行代码 (阶段4)
进入代码目录：
```bash
cd 4_code
# 如果是 Java 项目
mvn spring-boot:run
# 如果是 Python 项目
uvicorn main:app --reload
```
