│   └── sequences/          # 序列图存放
├── 4_code/                 # [阶段4] 代码工程
│   ├── src/                # 后端源码 (Java/Python)
│   └── tests/              # 单元测试

1.  **Prompt 归档：** 每次用 Kimi/ChatGPT 生成了满意的结果后，**必须**把 Prompt 保存到 `prompt/` 文件夹。这是我们实验报告的重要素材。
2.  **图片输出：** `.svg` 或 `.png` 图片不要散落在代码目录，统一由插件生成到 `out/` 目录，或者手动移动过去。

## 3. Git 简明协作指南 (新手必读)

鉴于大家刚接触 GitHub，建议使用 VS Code 内置的源代码管理工具，或者终端命令。

### 3.1 第一次获取项目
打开 VS Code 终端 (`` Ctrl+` ``)，输入：
```bash
git clone https://github.com/Euato-Key/UML-Application-System-analysis-and-design.git
```

### 3.2 每日工作流程 (保姆级步骤)

**第一步：开工前，先拉取最新代码 (Pull)**
*   防止覆盖别人的代码，每次写代码前**必须**先做这一步。
```bash
git pull
```

**第二步：修改与保存**
*   在本地编辑文件、写代码、画图。

**第三步：提交更改 (Stage & Commit)**
1.  点击 VS Code 左侧第三个图标（源代码管理）。
2.  点击 **+** 号（暂存所有更改）。
3.  在输入框写备注（如：“完成了扫码签到活动图”）。
4.  点击 **提交 (Commit)** 按钮。
*   *或者使用命令：*
    ```bash
    git add .
    git commit -m "你的备注信息"
    ```

**第四步：上传到云端 (Push)**
```bash
git push
```

### 3.3 遇到冲突怎么办？
如果 `git push` 报错，提示有冲突：
1.  先执行 `git pull`。
2.  VS Code 会在文件中标出冲突的地方（红色/绿色区域）。
3.  **不要慌**，和队友商量保留谁的代码，点击“采用当前更改”或“采用传入更改”。
4.  解决完所有冲突后，重新执行 **第三步** 和 **第四步**。

## 4. AI 辅助开发工具链

本实验鼓励使用 AI 提高效率。推荐使用 **AI IDE** 替代普通 VS Code，实现“对话即代码”。

### 4.1 推荐编辑器 (任选其一)
*   **Cursor / Trae / Antigravity:**
    *   这些是基于 VS Code 修改的 AI 编辑器，界面和 VS Code 一模一样，插件通用。
    *   **核心功能 (Cmd+K / Ctrl+K):** 在代码区按下快捷键，直接输入：“帮我生成一个 SpringBoot 控制器，包含增删改查”，它会直接写代码。
    *   **Chat 侧边栏:** 可以把报错信息直接贴给它，让它修 bug。

### 4.2 协作模式

如：

*   **林世颖 (开发):** 使用 Cursor/Trae 生成 Java/Python 代码。
*   **帅军锐 (设计):** 将生成的 PlantUML 代码贴入 AI IDE，让 AI 检查语法错误。

## 5. VS Code 必备插件及用法

请所有成员在 VS Code (或 Cursor) 中安装以下插件：

### 5.1 PlantUML (核心绘图工具)
*   **ID:** `jebbs.plantuml`
*   **用途:** 编写 `.puml` 文件并预览 UML 图。
*   **用法:**
    1.  打开 `.puml` 文件。
    2.  键盘按 `Alt + D` (Windows) 或 `Option + D` (Mac)。
    3.  右侧会弹出实时预览图。
*   **依赖:** 电脑需要安装 Java 环境。

### 5.2 Markdown PDF
*   **ID:** `yzane.markdown-pdf`
*   **用途:** 将我们的 `.md` 文档转为 PDF 方便提交报告。
*   **用法:**
    1.  打开 `.md` 文件。
    2.  右键点击编辑区 -> 选择 `Markdown PDF: Export (pdf)`。

### 5.3 Graphviz Preview (可选)
*   如果 PlantUML 预览渲染慢或报错，可以装这个插件辅助渲染。

## 6. 补充协作规范

1.  **文件命名法：**
    *   **序号引导：** 像 `01-访谈录音.md` 这样加序号，方便排序。
    *   **英文优先：** 代码文件和图片尽量用英文命名（如 `scan_qr_code.puml`），避免乱码。
2.  **沟通机制：**
    *   遇到 `git` 搞不定的红字报错，立即截图发微信群，不要强行操作。
    *   修改公共文件（如目录结构）前，在群里吼一声。
3.  **备份机制：**
    *   `prompt` 文件夹是我们最重要的资产之一，**千万不要删**。如果 AI 生成的代码跑不通，有 Prompt 就能随时重来。
